## Decisions

### Synthesis now captures two distinct diffs: uncommitted changes (git diff HEAD) and committed changes since the last synthesis commit

**Rationale:** Developers who commit frequently mid-session were losing committed work from the synthesis — the old single git diff HEAD only showed unstaged/staged changes, missing everything already committed. The two-diff approach ensures the full session is captured regardless of commit frequency.
**Files:** src/git.ts, src/synthesize.ts, prompts/synthesize.md

### Last synthesis anchor point is determined by finding the most recent commit that touched .lodestar.md via git log --follow

**Rationale:** Lodestar needs a reliable 'since when' marker without requiring users to tag commits or maintain a separate state file. The .lodestar.md file itself is the natural anchor — the commit that last updated it is when the previous synthesis happened.
**Files:** src/git.ts

### When no prior synthesis exists (no .lodestar.md or no commit touching it), fall back to showing the last 20 commits as context instead of an empty commitLog

**Rationale:** First-run experience: a brand new project has no anchor commit, so the committed diff would be empty. Showing recent commit log gives the AI useful context about the project's history to seed the first synthesis.
**Files:** src/git.ts

### Package changes diff is now computed against the last synthesis commit (sinceCommit) rather than always against HEAD

**Rationale:** Consistent with the two-diff approach — package.json changes should reflect what was added since the last synthesis, not just uncommitted package.json edits.
**Files:** src/git.ts

### GitSnapshot interface extended with committedDiff and commitLog fields

**Rationale:** Downstream consumers (synthesize.ts) need all three data streams — uncommitted diff, committed diff, and commit log — to produce a complete synthesis. Adding them to the interface keeps the contract explicit.
**Files:** src/git.ts

### synthesize.md prompt updated to explain the two-diff input structure to the AI

**Rationale:** The AI receiving the prompt needs to understand what each input section represents — without this framing, it might treat the committed diff and uncommitted diff as redundant or conflate them.
**Files:** prompts/synthesize.md

### projectRoot is now optional in MCP tools, defaulting to cwd

**Rationale:** The primary use case is an AI assistant calling lodestar_synthesize or lodestar_load from within a coding session in the project directory. Requiring an explicit path would add friction — defaulting to cwd matches natural usage.
**Files:** src/index.ts

### README.md added as the primary onboarding document covering installation, setup, and usage

**Rationale:** The project is public-facing (GitHub link present) and needs documentation for users discovering it cold. The README covers the full lifecycle: install from source, lodestar init wizard, CLI usage, and MCP integration.
**Files:** README.md

### lodestar init includes a first-synthesize step that retries on invalid directory input

**Rationale:** New users who haven't used the tool before should be able to run a synthesis immediately after setup to validate everything works. The retry-on-invalid-directory fix prevents the init flow from crashing when users mistype a path.
**Files:** src/init.ts

### Init flow skips API key prompt when an existing config is already valid

**Rationale:** Re-running lodestar init (e.g., to reconfigure tool integrations) should not force the user to re-enter an API key that already works. Only prompt for a key if the provider changed or the existing key is invalid.
**Files:** src/init.ts

### LODESTAR_FILENAME constant '.lodestar.md' extracted into src/git.ts rather than hardcoded in multiple places

**Rationale:** The filename is referenced in findLastSynthesisCommit and will likely be referenced elsewhere. A named constant prevents typos and makes future renames a single-point change.
**Files:** src/git.ts

## Patterns

- **Provider abstraction under src/providers/ directory — each provider gets its own file within the directory** — src/providers/
- **Entry points compile to dist/*.js — cli.ts → dist/cli.js, init.ts → dist/init.js, index.ts → dist/index.js** — package.json scripts and bin field
- **History stored in .lodestar.history/ at the working directory root, gitignored** — .gitignore, src/history.ts
- **Prompts stored as markdown files in prompts/ directory (e.g. prompts/synthesize.md) and loaded at runtime with template variable substitution ({{variable_name}} syntax)** — prompts/synthesize.md, src/synthesize.ts
- **Configuration separated into src/config.ts — not inlined into individual modules** — src/config.ts
- **The .lodestar.md file serves dual purpose: it's the human-readable context document AND the anchor marker for determining what has changed since the last synthesis** — src/git.ts, findLastSynthesisCommit()
- **GitResult discriminated union returned from captureGitSnapshot — callers check result type before accessing snapshot fields** — src/git.ts, src/synthesize.ts

## Rejected Approaches

### Using only git diff HEAD (uncommitted changes) as the sole input to synthesis

**Reason:** Developers who commit mid-session lose all committed work from the synthesis. A session with 5 commits and no uncommitted changes would produce an empty diff, generating a useless synthesis.
