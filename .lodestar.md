## Decisions

### Added a CLI entry point at src/cli.ts exposed as the 'lodestar' binary via package.json 'bin' field, compiling to dist/cli.js

**Rationale:** The project is being transformed from a library/script into an installable CLI tool. The bin field allows 'npx lodestar' or global install usage.
**Files:** package.json, src/cli.ts

### Added a separate init entry point at src/init.ts, accessible via 'npm run init' (node dist/init.js)

**Rationale:** Initialization flow is separated from the main CLI loop, suggesting a first-run setup experience distinct from normal operation.
**Files:** src/init.ts, package.json

### Deleted src/claude.ts and replaced with a src/providers/ directory structure

**Rationale:** The monolithic claude.ts was replaced with a provider abstraction layer to support multiple AI backends (Anthropic and OpenAI are both now in dependencies). This enables provider-agnostic operation.
**Files:** src/claude.ts, src/providers/

### Added src/config.ts for configuration management

**Rationale:** With multiple providers and API keys, a dedicated config module is needed to manage provider selection, API keys, and user preferences — likely reading from .env or a config file.
**Files:** src/config.ts

### Added src/history.ts for session history management

**Rationale:** The .lodestar.history/ directory was added to .gitignore simultaneously, indicating history is stored locally on disk and excluded from version control.
**Files:** src/history.ts, .gitignore

### History directory (.lodestar.history/) is gitignored

**Rationale:** Session history is user-local data, not project source — it should not be committed to the repository.
**Files:** .gitignore

### Both @anthropic-ai/sdk and openai packages added as runtime dependencies simultaneously

**Rationale:** The provider abstraction (src/providers/) supports both Anthropic Claude and OpenAI GPT as backends, letting users choose their preferred provider.
**Files:** package.json, src/providers/

### Used @inquirer/prompts for interactive CLI input rather than readline or prompts

**Rationale:** @inquirer/prompts provides polished interactive prompts (select, confirm, input, checkbox) with good TypeScript support, appropriate for a CLI tool that guides users through setup and operations.
**Files:** package.json, src/cli.ts, src/init.ts

### Modified src/synthesize.ts, src/diff.ts, src/git.ts, src/load.ts, and src/index.ts — all core pipeline files

**Rationale:** The core synthesis pipeline was refactored, likely to support the new provider abstraction and CLI entrypoint, though the specific changes are not fully visible due to diff truncation.
**Files:** src/synthesize.ts, src/diff.ts, src/git.ts, src/load.ts, src/index.ts

### Added 'open' package as a runtime dependency

**Rationale:** Likely used to open browser URLs from the CLI — possibly for OAuth flows (API key setup), documentation links, or opening generated output files.
**Files:** package.json

### Added CLAUDE.md at project root

**Rationale:** CLAUDE.md is the standard file for giving Claude AI context about a project when used with Claude Code or similar tools. This is the project telling Claude how to work with it — meta and appropriate for a tool called lodestar.
**Files:** CLAUDE.md

### Updated prompts/synthesize.md

**Rationale:** The synthesis prompt was modified, likely to accommodate multi-provider output formats or to refine the context document structure being generated.
**Files:** prompts/synthesize.md

## Patterns

- **Provider abstraction under src/providers/ directory — each provider gets its own file within the directory** — src/providers/
- **Entry points compile to dist/*.js — cli.ts → dist/cli.js, init.ts → dist/init.js, index.ts → dist/index.js** — package.json scripts and bin field
- **History stored in .lodestar.history/ at the working directory root, gitignored** — .gitignore, src/history.ts
- **Prompts stored as markdown files in prompts/ directory (e.g. prompts/synthesize.md) and loaded at runtime** — prompts/synthesize.md, src/load.ts
- **Configuration separated into src/config.ts — not inlined into individual modules** — src/config.ts

## Dependencies

- **@anthropic-ai/sdk** — Anthropic Claude API client — one of the supported AI provider backends in src/providers/
- **openai** — OpenAI API client — the second supported AI provider backend in src/providers/
- **@inquirer/prompts** — Interactive CLI prompts (select, confirm, input, etc.) used in the CLI and init flows
- **@modelcontextprotocol/sdk** — MCP (Model Context Protocol) SDK — was already in the project's purpose based on package keywords, now explicitly installed as a runtime dep for serving or consuming MCP tools
- **open** — Opens URLs or files in the default browser/application — likely used for OAuth key setup flows or opening output
- **simple-git** — Git operations — used in src/git.ts to read git diff, status, and log data that feeds the synthesis pipeline
